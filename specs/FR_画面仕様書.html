入力項目定義 Form Fields

項目名入力方式バリデーション種別

始業点検 トグルスイッチ（OK / NG） — 必須
安全確認 トグルスイッチ（OK / NG） — 必須
設備異常 チェックボックス チェック時は異常内容テキストエリアが展開 任意
作業メモ テキストエリア（最大500文字） 500文字以内 任意

ボタン名動作有効条件

基本情報 Overview

画面IDSCR-003
画面名完了確認画面
遷移元SCR-002（作業日報入力）/ SCR-002b（点検記録入力）の「確認画面へ」ボタン

表示要素 UI Elements

要素名内容種別備考
基本情報カード 担当工程・作業日・製品品番をサマリ表示 自動 前画面の入力値をそのまま表示
生産実績カード 製造数量・不良数量・不良コードを表示 自動 不良数量 > 0 の場合はオレンジ色で強調表示
点検チェックカード 各点検項目の結果（OK/NG）を表示 自動 NG項目は赤色で強調表示
送信ボタン 「この内容で送信する」。タップで確認ダイアログを表示後、APIへPOST送信。 必須 送信中はローディングスピナーを表示しボタンを無効化
修正するボタン 前の入力画面へ戻る（入力値は保持） 必須 —

送信処理フロー Submit Flow

状態動作UI表現
送信前「この内容で送信する」ボタンを表示通常状態
確認ダイアログ「送信しますか？」のモーダルダイアログを表示「確定」「キャンセル」の2択
送信中APIへPOST。ボタンをスピナーに変更し、操作を無効化ローディングインジケーター
送信成功完了モーダルを3秒表示後、自動的にSCR-001へ遷移緑チェックアイコン・完了メッセージ
送信失敗（ネットワーク）画面上部にエラーバナーを表示。「リトライ」ボタンを表示。赤バナー・リトライボタン
送信失敗（APIエラー）エラーコードとメッセージを表示。ヘルプデスク連絡先を案内。赤バナー・エラーコード表示
オフライン送信端末に暗号化保存。「オフライン保存しました。接続回復後に自動送信します。」を表示してSCR-001へ遷移黄色バナー・自動送信アイコン
💡 送信ボタンは二重タップによる重複送信を防ぐため、タップ後は即時無効化すること。送信完了後のホーム遷移は自動（3秒後）とするが、「ホームへ戻る」ボタンでも手動遷移可能とする。
